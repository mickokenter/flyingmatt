Introduction

This code sample shows how to retrieve a map image from Bing Maps that tightly fits to a specified bounding box. The Bing Maps REST Imagery service does allow you to retrieve an image that contains a bounding box, but the image will likely not tightly fit to the bounding box due to maintaining the aspect ratio of the specified map dimensions and also due to the fact that the service uses one of 19 zoom levels. This code samples takes in a bounding box and based on that calculates the center point and zoom level that would contain the specified bounding box in the largest map image size supported by the service. From there the code calculates the pixel coordinates of the bounding box corners and uses these to calculate the pixel size of the map image that would fit title around the bounding box. Putting this all together, the center, zoom, and calculated map dimensions are used to retrieve an image that fits around the specified bounding box tightly. A lot of the calculations used in this code is based on the tile system math used by Bing Maps that is documented here and the map view calcuations done here.

Building the Sample

To get this code sample to work you will need to go into the MainWindow.xaml.cs file and insert your Bing Maps key at the top where it says YOUR_BING_MAPS_KEY. If you do not have a Bing Maps key you can get one here.
